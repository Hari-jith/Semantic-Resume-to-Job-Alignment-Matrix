# Semantic Resume-to-Job Alignment Matrix

> A Transformer-based semantic resume screening system that matches resumes with job descriptions using dense vector embeddings, FAISS semantic search, and explainable ATS scoring — now with an interactive Streamlit app.

---

## Overview

Traditional Applicant Tracking Systems (ATS) primarily rely on keyword matching when screening resumes. As a result, semantically similar skills may not be recognized if the exact keywords differ.

For example,

Job Description:

> Python

Resume:

> PyTorch

Although PyTorch is built on Python, many keyword-based ATS systems fail to recognize this relationship.

This project addresses that limitation using Transformer-based semantic embeddings generated by **Sentence Transformers (all-mpnet-base-v2)**.

Instead of searching for exact keywords, the system compares the semantic meaning of resume content with job requirements.

## Transformer Model

### Embedding Model

```
sentence-transformers/all-mpnet-base-v2
```

Why MPNet?

- Excellent semantic understanding
- Produces 768-dimensional dense vectors
- Fast inference
- Open source
- Works efficiently on free Kaggle GPU environments

## Project Pipeline

### 1. Data Cleaning

- Remove duplicates
- Normalize text
- Handle missing values

---

### 2. Semantic Chunking

Long resumes are divided into overlapping semantic chunks.

Example

```
Chunk 1

Java Spring Boot
Hibernate
REST APIs

-------------------

Chunk 2

SQL
MySQL
Database Design
```

---

### 3. Embedding Generation

Each chunk is converted into a

```
768-dimensional semantic vector
```

using MPNet.

---

### 4. Semantic Retrieval

Instead of keyword search,

FAISS retrieves the most semantically similar Job Description chunks.

---

### 5. Alignment Matrix

Each Resume Chunk is aligned with its closest Job Description chunks.

Example

| Resume Chunk | Job Description Chunk | Similarity |
|--------------|----------------------|-----------:|
| Spring Boot | Java Backend | 0.91 |
| SQL | Database Design | 0.88 |

---

### 6. ATS Score

Similarity scores are aggregated into an overall ATS compatibility score.

## Streamlit Application

An interactive Streamlit app is included so a user can upload their own resume and job description and get a live semantic alignment report, without touching the notebook.

### App Capabilities

- Upload resume and job description as **PDF, DOCX, or TXT** (or paste text directly)
- Adjustable chunking parameters (chunk size, overlap, Top-K matches) from the sidebar
- Live text cleaning, chunking, and MPNet embedding generation
- FAISS-based semantic search between resume chunks and JD chunks
- ATS Score, Grade, Mean Similarity, and Coverage displayed as summary metrics
- Explainable Report tab showing best JD match per resume chunk
- Strong Matches tab (similarity ≥ 0.80)
- Weak Matches tab (similarity < 0.60)
- Full Alignment Matrix view (all Top-K matches, not just the best one)

### Project Structure

```
resume_ats_app/
├── app.py
├── requirements.txt
└── utils/
    ├── __init__.py
    ├── text_utils.py         # cleaning, title normalization, chunking
    ├── embedding_utils.py    # MPNet model loading + embedding generation
    ├── scoring_utils.py      # FAISS index, alignment matrix, ATS scoring
    └── file_utils.py         # PDF / DOCX / TXT text extraction
```

### Running the App

```bash
pip install -r requirements.txt
streamlit run app.py
```

The app will open in your browser. Upload or paste a resume and a job description, adjust chunking parameters if needed, and click **Run Semantic Alignment**.
